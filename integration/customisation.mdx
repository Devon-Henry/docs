## Overview

The Ragou chat widget offers extensive customization options to seamlessly integrate with your website's design and branding. Configure colors, icons, animations, and more through simple HTML attributes.

## Configuration Properties

### Required Properties

<AccordionGroup>
  <Accordion icon="key" title="projectId">
    **Type:** `string`  
    **Required:** Yes
    
    Your unique Ragou project identifier. This connects the widget to your specific project and data.
    
    ```html
    <chat-widget projectId="your-project-id"></chat-widget>
    ```
    
    <Note>Find your Project ID in your [Ragou dashboard](https://app.ragou.net/home/teams/projects).</Note>
  </Accordion>
</AccordionGroup>

### Visual Customization

<AccordionGroup>
  <Accordion icon="palette" title="color">
    **Type:** `string`  
    **Default:** `#007bff`  
    **Format:** Hex color, RGB, HSL, or named CSS colors
    
    Sets the primary color for the chat widget button and header background.
    
    ```html
    <!-- Hex color -->
    <chat-widget color="#8B5CF6" projectId="your-project-id"></chat-widget>
    
    <!-- RGB color -->
    <chat-widget color="rgb(139, 92, 246)" projectId="your-project-id"></chat-widget>
    
    <!-- Named color -->
    <chat-widget color="purple" projectId="your-project-id"></chat-widget>
    ```
  </Accordion>

  <Accordion icon="brush" title="theme">
    **Type:** `string`  
    **Default:** `system`  
    **Options:** `light`, `dark`, `system`
    
    Controls the overall theme of the chat interface. System theme automatically adapts to user's device preference.
    
    ```html
    <!-- Light theme -->
    <chat-widget theme="light" projectId="your-project-id"></chat-widget>
    
    <!-- Dark theme -->
    <chat-widget theme="dark" projectId="your-project-id"></chat-widget>
    
    <!-- System theme (follows device preference) -->
    <chat-widget theme="system" projectId="your-project-id"></chat-widget>
    ```
  </Accordion>

  <Accordion icon="paintbrush" title="primaryColor">
    **Type:** `string`  
    **Default:** `#007bff`  
    **Format:** 6-character hex color code (without #)
    
    Sets the primary color for chat elements like user message bubbles and submit buttons inside the chat interface.
    
    ```html
    <!-- Purple primary color -->
    <chat-widget primaryColor="8B5CF6" projectId="your-project-id"></chat-widget>
    
    <!-- Green primary color -->
    <chat-widget primaryColor="10B981" projectId="your-project-id"></chat-widget>
    
    <!-- Custom brand color -->
    <chat-widget primaryColor="FF6B6B" projectId="your-project-id"></chat-widget>
    ```
    
    <Note>Use 6-character hex codes without the # symbol (e.g., "8B5CF6" not "#8B5CF6")</Note>
  </Accordion>

  <Accordion icon="font" title="textColor">
    **Type:** `string`  
    **Default:** `ffffff` (white)  
    **Format:** 6-character hex color code (without #)
    
    Sets the text color for user messages when using a custom primary color.
    
    ```html
    <!-- White text on colored background -->
    <chat-widget 
        primaryColor="8B5CF6" 
        textColor="ffffff" 
        projectId="your-project-id">
    </chat-widget>
    
    <!-- Black text for light backgrounds -->
    <chat-widget 
        primaryColor="F3F4F6" 
        textColor="000000" 
        projectId="your-project-id">
    </chat-widget>
    ```
  </Accordion>

  <Accordion icon="heading" title="titleColor">
    **Type:** `string`  
    **Default:** `ffffff` (white)  
    **Format:** 6-character hex color code (without #)
    
    Sets the color of the title text and close button in the chat window header.
    
    ```html
    <!-- Custom title color -->
    <chat-widget 
        titleColor="FFD700" 
        title="Support Chat"
        projectId="your-project-id">
    </chat-widget>
    
    <!-- Dark title for light headers -->
    <chat-widget 
        color="#F8F9FA"
        titleColor="374151" 
        projectId="your-project-id">
    </chat-widget>
    ```
  </Accordion>
  
  <Accordion icon="image" title="icon">
    **Type:** `string`  
    **Default:** Default chat bubble SVG  
    **Format:** URL, Data URL, SVG markup, or emoji
    
    Customizes the icon displayed on the chat widget button.
    
    ```html
    <!-- Image URL -->
    <chat-widget icon="https://example.com/chat-icon.png" projectId="your-project-id"></chat-widget>
    
    <!-- Relative path -->
    <chat-widget icon="/images/support-icon.svg" projectId="your-project-id"></chat-widget>
    
    <!-- Emoji -->
    <chat-widget icon="ðŸ’¬" projectId="your-project-id"></chat-widget>
    
    <!-- SVG markup -->
    <chat-widget icon='<svg viewBox="0 0 24 24"><path d="M12 2C6.48 2 2 6.48 2 12s4.48 10 10 10 10-4.48 10-10S17.52 2 12 2z"/></svg>' projectId="your-project-id"></chat-widget>
    ```
    
    <Tip>For best results, use square images (1:1 aspect ratio) at 28x28 pixels or larger.</Tip>
  </Accordion>
  
  <Accordion icon="text" title="title">
    **Type:** `string`  
    **Default:** `"Chat Support"`
    
    Sets the title displayed in the chat window header.
    
    ```html
    <chat-widget title="Help & Support" projectId="your-project-id"></chat-widget>
    
    <chat-widget title="Ask Ragou" projectId="your-project-id"></chat-widget>
    ```
  </Accordion>
</AccordionGroup>

### Behavior Options

<AccordionGroup>
  <Accordion icon="radio" title="pulsing">
    **Type:** `boolean`  
    **Default:** `true`
    
    Controls whether the chat widget button has a pulsing animation to draw attention.
    
    ```html
    <!-- Enable pulsing (default) -->
    <chat-widget pulsing="true" projectId="your-project-id"></chat-widget>
    
    <!-- Disable pulsing -->
    <chat-widget pulsing="false" projectId="your-project-id"></chat-widget>
    ```
    
    <Note>The pulsing animation helps draw user attention to the chat widget.</Note>
  </Accordion>
  
  <Accordion icon="window-maximize" title="isOpen">
    **Type:** `boolean`  
    **Default:** `false`
    
    Controls the initial state of the chat window (opened or closed).
    
    ```html
    <!-- Start with chat window closed (default) -->
    <chat-widget isOpen="false" projectId="your-project-id"></chat-widget>
    
    <!-- Start with chat window open -->
    <chat-widget isOpen="true" projectId="your-project-id"></chat-widget>
    ```
  </Accordion>
</AccordionGroup>

## Advanced Theming

The chat widget automatically passes theme and color customizations to the internal chat interface through URL parameters. This ensures a consistent experience between the widget button and the chat window.

### Parameter Passing

When you set properties on the widget, they are automatically passed to the chat interface:

```html
<chat-widget 
    theme="dark"
    primaryColor="8B5CF6"
    textColor="ffffff"
    projectId="your-project-id">
</chat-widget>
```

This generates a chat URL with parameters like:
```
https://chat.ragou.net/?projectId=your-project-id&theme=dark&primaryColor=8B5CF6&textColor=ffffff
```

### Theme Inheritance

- **System Theme**: Automatically detects user's device preference
- **Light Theme**: Forces light mode regardless of device setting  
- **Dark Theme**: Forces dark mode regardless of device setting
- **Color Consistency**: Primary and text colors are applied to user messages and interactive elements

```html
        title="AI Assistant"

        primaryColor="8B5CF6"
        textColor="ffffff"
        titleColor="ffffff"

        pulsing="true"
        isOpen="false">
    </chat-widget>
</body>
</html>
```

## Best Practices

<AccordionGroup>
  <Accordion icon="lightbulb" title="Color Selection">
    - Use colors that contrast well with white text for readability
    - Consider your brand's primary or accent colors
    - Test colors in both light and dark environments
    - Ensure sufficient contrast for accessibility (WCAG AA compliance)
  </Accordion>

  <Accordion icon="palette" title="Color Coordination">
    When using multiple color properties, ensure they work well together:
    
    - **Widget Button (`color`)**: Should be prominent and match your brand
    - **Primary Color (`primaryColor`)**: Used for user messages and interactive elements
    - **Text Color (`textColor`)**: Must contrast well with the primary color
    - **Title Color (`titleColor`)**: Should be readable against the widget button color
    
    ```html
    <!-- Good color coordination -->
    <chat-widget 
        color="#1E40AF"      <!-- Blue widget button -->
        primaryColor="3B82F6" <!-- Lighter blue for messages -->
        textColor="ffffff"    <!-- White text on blue -->
        titleColor="ffffff"   <!-- White title on blue header -->
        projectId="your-project-id">
    </chat-widget>
    ```
  </Accordion>

  <Accordion icon="moon" title="Theme Considerations">
    - Use `theme="system"` to automatically adapt to user preferences
    - Consider providing both light and dark color schemes
    - Test your widget in all theme modes before deployment
    - Dark themes work well with lighter accent colors
  </Accordion>
  
  <Accordion icon="image" title="Icon Guidelines">
    - Use square images for best results
    - Optimize file sizes for faster loading
    - Ensure icons are legible at 28x28 pixels
    - Consider using SVG format for crisp scaling
  </Accordion>
  
  <Accordion icon="mobile" title="Mobile Considerations">
    - Test the widget on various screen sizes
    - Ensure the chat window doesn't obstruct important content
    - Consider disabling pulsing on mobile to reduce battery usage
  </Accordion>
</AccordionGroup>

## Troubleshooting

<Warning>
  If your widget isn't appearing, check:
  - The script is loaded before the widget element
  - There are no JavaScript console errors
  - The widget element is properly closed: `</chat-widget>`
</Warning>

<Tip>
  You can inspect the widget element in your browser's developer tools to verify that properties are being applied correctly.
</Tip>
